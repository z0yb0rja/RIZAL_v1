Heavy Concurrent Traffic Recommendations (VALID8)

1. Stateless API replicas + load balancer
- Run multiple FastAPI instances behind Nginx/Traefik/ALB.
- Use a production server (gunicorn + uvicorn workers), not --reload.

2. Connection pooling layer
- Add PgBouncer in front of Postgres.
- Tune SQLAlchemy pool and Postgres max_connections.

3. Redis for hot reads and rate limiting
- Cache dashboard counts and common read queries.
- Use Redis-backed rate limiting and short-lived session/cache data.

4. Queue long-running work
- Keep request/response path fast.
- Move email/SMS/import/report generation to Celery workers.

5. DB tuning and query hygiene
- Ensure indexes for school_id, created_at, and frequent FK filters.
- Use pagination for large lists.
- Avoid N+1 queries.
- Add read replicas if read-heavy.

6. Autoscaling + observability
- Scale API and workers independently.
- Add metrics, logs, tracing, and alerts.
- Run load tests (k6/Locust) before production go-live.

Highest-impact first for this project
- Switch to production app server + multiple replicas.
- Add PgBouncer.
- Add Redis caching for heavy list/report endpoints.
- Run load tests and tune based on bottlenecks.
